以下是「個人新創作弊表：零前期成本 SaaS 堆棧的真實結構」的語音版本。

二零二六年三月初，一張由 RoundtableSpace 帳號分享的「個人創辦人作弊表」在推特上廣泛流傳。這張圖列出十個工具的組合，聲稱可以讓單一創辦人在收到第一筆營收之前，每月基礎設施成本為零美元。這個說法引起大量討論，也促使許多人重新評估二零二六年個人建構 SaaS 產品的實際門檻。

背景方面，過去五年，SaaS 創業的固定成本結構發生了根本性轉變。二零一九年，一個典型的最小可行產品上線至少需要亞馬遜雲的虛擬機、資料庫、加上至少一名後端工程師的薪資。到了二零二六年，這些依賴幾乎全部可以用免費層服務替換，而 AI 輔助編碼工具讓單人開發的生產力接近小型團隊的水準。這個堆棧的核心邏輯不是「便宜」，而是「預收入階段零固定支出」。Stripe 只在收到付款後才收費，所以零工資燃燒的說法在早期具有字面意義上的準確性。

接下來介紹各工具。

第一個是 n8n，用於工作流程自動化。n8n 是這套堆棧裡最有彈性但也最需要技術決策的工具，定位是個人創辦人的後端黏合劑。它可以在用戶完成註冊後自動觸發歡迎郵件、在 Stripe 收到付款後更新資料庫狀態、監聽外部平台關鍵字並路由到對應流程。雲端版每月二十美元但限制在兩千五百次工作流執行，對有週期性任務的產品很快見頂。自架版本免費，但需要維護一台伺服器。

第二個是 Supabase，作為後端核心。Supabase 是基於 PostgreSQL 的開源後端平台，免費層包含五百兆位元組資料庫、五萬月活用戶的身份驗證、以及自動產生的 REST 和 GraphQL API。最主要的限制是閒置一週後自動暫停，對沒有穩定日常流量的工具構成生產風險。

第三個是 Cursor 加上 Claude，組成 AI 輔助開發組合。Cursor 是基於 VS Code 的 AI 原生代碼編輯器，深度整合 Claude 模型。這個組合最有效的場景是你知道要建什麼、能描述清楚業務邏輯，但對特定技術實作方式不確定。Cursor 免費層有限制，認真開發幾乎需要付費的 Pro 版。

第四個是 Vercel，負責部署平台。Vercel 的免費方案提供自動持續部署和全球內容分發網路。限制在於伺服器函式每次執行有十秒逾時，需要較長執行時間的 API 呼叫會提早碰到這個限制。

第五個是 Stripe，處理支付。Stripe 按交易收費，沒有月費，在有實際收入之前成本為零。

第六個是 Resend，提供電子郵件服務。免費方案每月三千封郵件，對早期 SaaS 的交易郵件需求通常足夠。

第七個是 Framer，用於建立登陸頁面。免費方案限 Framer 子網域，連結自訂網域需要每月十美元。優勢是可以快速迭代行銷頁面，不用動主應用的程式碼。

第八個是 PostHog，提供產品分析。免費方案每月一百萬事件，整合了事件追蹤、會話回放和 A/B 測試，建議從第一天就安裝。

第九個是 Cloudflare，負責安全與基礎設施。免費層提供 DDoS 緩解、SSL 加密和全球內容分發，對個人創辦人主要解決意外流量湧入的風險。

關於真實成本結構，這套工具在嚴格定義下確實可以接近零成本。但一個實際要對外的最小可行產品，考慮 Framer 自訂網域十美元、n8n 雲端版二十美元、Cursor Pro 二十美元，加上網域費用，每月大約五十一美元。更準確的說法是預收入階段固定成本可以壓縮到每月零到五十一美元，相比傳統 SaaS 創業所需的技術人力成本仍然是數量級的差異。

限制與風險方面，Supabase 的閒置暫停最容易被忽略。Vercel 的十秒超時限制了業務邏輯的複雜度。供應商鎖定是最深層的結構性問題，十個工具中九個是外部服務，資料分散在不同供應商，遷移成本不低。定價變動風險也不可忽略，Vercel 和 Supabase 都曾調整過定價結構。

這套堆棧適合對 Next.js 生態系熟悉、產品模式是訂閱制 SaaS、優先驗證市場需求的創辦人。面向 B2B 小型 SaaS、API 包裝產品、以及行業特定自動化工具，匹配度最高。

最後，這張作弊表的真正價值不是零成本，而是把基礎設施決策的成本從前期轉移到後期。你不需要在有第一個付費用戶之前就處理 DevOps、資料庫維運、郵件服務配置。這些決策都推遲到你確認有人願意付錢的時候。當基礎設施成本接近零時，創辦人可以同時運行多個實驗，快速淘汰沒有市場需求的方向，而不需要在每次嘗試前做融資決策。
